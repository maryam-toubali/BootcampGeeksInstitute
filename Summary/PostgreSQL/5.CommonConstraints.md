| Constraint    | Example                            | What It Does                                                    | When to Use It                                                             |
| ------------- | ---------------------------------- | --------------------------------------------------------------- | -------------------------------------------------------------------------- |
| `PRIMARY KEY` | `id SERIAL PRIMARY KEY`            | Uniquely identifies each row. Cannot be NULL.                   | Always use for the main identifier of a row (e.g., user ID, product ID).   |
| `UNIQUE`      | `email VARCHAR(100) UNIQUE`        | Ensures all values in the column are unique.                    | Use when no duplicates are allowed — e.g., usernames, emails.              |
| `NOT NULL`    | `name TEXT NOT NULL`               | Column must have a value. Cannot be empty.                      | Use when data is required, like `name`, `price`, `quantity`.               |
| `CHECK`       | `CHECK (price > 0)`                | Restricts column values to specific rules.                      | For business rules, like positive prices or valid ranges.                  |
| `DEFAULT`     | `in_stock BOOLEAN DEFAULT TRUE`    | Auto-fills column with a default value if none is provided.     | Great for booleans, timestamps, or typical status flags.                   |
| `FOREIGN KEY` | `user_id INT REFERENCES users(id)` | Links to a row in another table. Enforces relational integrity. | Use in related tables to connect data — e.g., `orders.user_id → users.id`. |
