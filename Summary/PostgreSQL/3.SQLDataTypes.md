

## 📘 **SQL Data Types — Full Summary**

Understanding **data types** is essential because **every column** in a table must have one — it defines **what kind of data** is allowed in that column.

### 🔢 1. Numeric Types

| Data Type         | Example    | What It Stores               | When to Use It                                                    |
| ----------------- | ---------- | ---------------------------- | ----------------------------------------------------------------- |
| `SMALLINT`        | 25         | Small integers               | For small numbers like age, rating (–32,768 to 32,767)            |
| `INTEGER` / `INT` | 154        | Regular integers             | General-purpose numbers (–2 billion to 2 billion)                 |
| `BIGINT`          | 9000000000 | Large integers               | Very big numbers like population or bank accounts                 |
| `SERIAL`          | (auto)     | Auto-increment integer       | For primary keys like `id`. PostgreSQL handles auto-incrementing. |
| `NUMERIC(10,2)`   | 105.75     | Precise decimal (money, tax) | When exact decimal math is required (e.g., financial apps)        |
| `REAL` / `FLOAT`  | 3.14       | Approximate decimals         | For scientific or less precise data                               |

---

### 🧾 2. Text Types

| Data Type    | Example       | What It Stores       | When to Use It                                                    |
| ------------ | ------------- | -------------------- | ----------------------------------------------------------------- |
| `CHAR(n)`    | 'YES     '    | Fixed-length strings | Rare — stores exactly `n` characters, padded with spaces          |
| `VARCHAR(n)` | 'Hello'       | Variable strings     | Use when you know the max length (e.g., `VARCHAR(100)` for names) |
| `TEXT`       | Long messages | Unlimited text       | Use for long text like comments or descriptions                   |

---

### ✅ 3. Boolean Type

| Data Type | Example | What It Stores | When to Use It                            |
| --------- | ------- | -------------- | ----------------------------------------- |
| `BOOLEAN` | `TRUE`  | True or False  | For switches: is\_active, is\_admin, etc. |

---

### 📅 4. Date & Time Types

| Data Type     | Example                  | What It Stores         | When to Use It                          |
| ------------- | ------------------------ | ---------------------- | --------------------------------------- |
| `DATE`        | `2025-05-12`             | Calendar date          | For birthdates, deadlines               |
| `TIME`        | `14:30:00`               | Time of day            | When you only care about time           |
| `TIMESTAMP`   | `2025-05-12 14:30:00`    | Date + time            | For logs, creation times                |
| `TIMESTAMPTZ` | Timestamp with time zone | Date, time & time zone | For global apps where time zones matter |

---

## 🛠️ Example: Practice Table with All Data Types

```sql
CREATE TABLE products (
  id SERIAL PRIMARY KEY,             -- Auto ID
  name VARCHAR(100) NOT NULL,        -- Name of product
  description TEXT,                  -- Full description
  price NUMERIC(10,2),               -- Precise price
  in_stock BOOLEAN DEFAULT TRUE,     -- Available or not
  created_at TIMESTAMP DEFAULT now()-- Time of creation
);
```
